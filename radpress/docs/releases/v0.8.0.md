# Batoi Press 0.8.0

Batoi Press 0.8.0 completes the first author-governance pass by enforcing post ownership inside the post workflow.

## What Changed

- Authors now see only posts assigned to their username.
- Authors cannot open edit screens for posts authored by other users.
- Authors cannot save changes against posts authored by other users.
- Blocked post ownership attempts return a 403 page.
- Blocked ownership attempts are recorded in the audit log as `post.access_blocked`.
- Added focused post ownership regression tests.

## Role Access

- Owner, admin, and editor users can manage all posts.
- Author users can create posts and manage posts where `author` matches their username.
- Viewer users cannot manage posts.

## Compatibility

This release is backward compatible with the v0.7.x content format, theme templates, public routes, installer layout, release manifest structure, and update package model.

Existing posts without an `author` value are not manageable by author-role users until an owner, admin, or editor updates ownership through the normal post save flow.

## Release Checklist

- Update `radpress/config/update.json` to `0.8.0`.
- Run `php radpress/tests/smoke.php`.
- Run `php radpress/tests/update_runner.php`.
- Run `php radpress/tests/static_export.php`.
- Run `php radpress/tests/role_access.php`.
- Run `php radpress/tests/post_ownership.php`.
- Build the package with `php tools/build-release.php`.
- Generate the manifest with `php tools/generate-release-manifest.php`.
- Verify release outputs with `php tools/verify-release-artifacts.php`.
- Publish `dist/latest.json` and `dist/batoi-press-0.8.0.zip` to `https://www.batoi.com/pub/press/`.

## Follow-Up

Continue toward `1.0.0` by deciding whether pages need ownership controls and whether editorial approval states are required before stable release.
