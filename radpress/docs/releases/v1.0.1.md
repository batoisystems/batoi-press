# Batoi Press 1.0.1

Batoi Press 1.0.1 is a patch release for issues raised during testing of the stable admin and public-site workflows.

## Fixes

- Improved the filter controls on Posts, Pages, and Media so they align with the admin console theme.
- Allowed safe form controls in page and post Body HTML, including required text inputs and submit buttons.
- Prevented embedded Body HTML form controls from blocking admin page/post save submission.
- Preserved sanitizer protection for script, event handler, style, and unsafe form action URLs.
- Improved theme template save validation so template syntax-check workspace issues return clear admin errors.
- Improved static export failure handling so generation errors are reported in the admin interface.
- Connected the public default theme header to the saved Main Menu configuration.
- Displayed the current favicon in Settings, falling back to the bundled Batoi Press icon when no custom favicon is configured.

## Compatibility

This release is backward compatible with the `1.0.0` content format, theme templates, public routes, installer layout, release manifest structure, and update package model.

## Verification

- `php -l` on changed PHP files
- `php radpress/tests/smoke.php`
- `php radpress/tests/static_export.php`

- Targeted theme template validation check

- `php tools/verify-release-artifacts.php`

## Publication

Publish `dist/latest.json` and `dist/batoi-press-1.0.1.zip` to the public Batoi Press release directory after local verification.
