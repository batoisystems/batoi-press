# Batoi Press 0.7.0

Batoi Press 0.7.0 enforces the admin roles already stored in user records, making route access and sidebar navigation role-aware.

## What Changed

- Added centralized admin route access rules.
- Blocked unauthorized admin requests with a 403 page.
- Recorded blocked admin route attempts in the audit log.
- Hid sidebar and topbar update links unavailable to the current user's role.
- Added focused role access regression tests.

## Role Access

- Owner: full installation governance and recovery access.
- Admin: full administration, with owner safeguards still enforced by user lifecycle controls.
- Editor: pages, posts, media, menus, and Batoi AIF assist.
- Author: post workflow routes.
- Viewer: dashboard-only access.

## Compatibility

This release is backward compatible with the v0.6.x content format, theme templates, public routes, installer layout, release manifest structure, and update package model.

Existing users without a recognized role fall back to viewer access until an owner or admin updates the role.

## Release Checklist

- Update `radpress/config/update.json` to `0.7.0`.
- Run `php radpress/tests/smoke.php`.
- Run `php radpress/tests/update_runner.php`.
- Run `php radpress/tests/static_export.php`.
- Run `php radpress/tests/role_access.php`.
- Build the package with `php tools/build-release.php`.
- Generate the manifest with `php tools/generate-release-manifest.php`.
- Verify release outputs with `php tools/verify-release-artifacts.php`.
- Publish `dist/latest.json` and `dist/batoi-press-0.7.0.zip` to `https://www.batoi.com/pub/press/`.
