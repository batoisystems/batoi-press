# Batoi Press 1.4.3

Batoi Press 1.4.3 fixes uploaded branding previews when the application is installed below a URL subdirectory.

## Fixes

- Prevents the installation base path from being added twice to uploaded brand-logo preview URLs.
- Prevents the installation base path from being added twice to uploaded favicon preview URLs.
- Stops valid custom favicons from incorrectly falling back to the Batoi Press icon in Settings.
- Keeps default branding asset previews correctly localized for root and subdirectory installations.

## Compatibility

Existing uploaded logo and favicon files and their stored configuration paths remain unchanged. The update only corrects URL rendering in the Admin Settings preview.

## Verification

All regression suites passed. Chrome verification on a subdirectory test installation confirmed that both uploaded previews returned the intended image, used a single localized base path, and rendered with valid natural dimensions.
