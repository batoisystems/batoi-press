# Batoi Press 0.6.0

Batoi Press 0.6.0 strengthens static export handoff for pre-1.0 production pilots by verifying generated static packages and including media deployment guidance in every export ZIP.

## What Changed

- Static exports now include `media/README.txt` with deployment guidance for referenced uploaded media.
- Generated static export ZIP files are verified immediately after creation.
- Verification checks required page, post, blog, sitemap, feed, and media guidance entries.
- Verification rejects unsafe archive paths and private/admin/runtime paths.
- The Static Export completion screen now shows package verification status, errors, and warnings.
- A focused static export regression test covers package contents and verification results.

## Compatibility

This release is backward compatible with the v0.5.x content format, theme templates, public routes, installer layout, release manifest structure, and update package model.

Static exports still focus on published generated output. Uploaded media files are not copied automatically; copy required files from `radpress/content/media/` to the target site's `media/` directory when published content references `/media/{file}`.

## Release Checklist

- Update `radpress/config/update.json` to `0.6.0`.
- Run `php radpress/tests/smoke.php`.
- Run `php radpress/tests/update_runner.php`.
- Run `php radpress/tests/static_export.php`.
- Build the package with `php tools/build-release.php`.
- Generate the manifest with `php tools/generate-release-manifest.php`.
- Verify release outputs with `php tools/verify-release-artifacts.php`.
- Publish `dist/latest.json` and `dist/batoi-press-0.6.0.zip` to `https://www.batoi.com/pub/press/`.

## Follow-Up

Continue toward `1.0.0` by browser-reviewing the full admin console, deciding whether static export should optionally copy media files, and documenting final production deployment expectations.
