# Batoi Press 1.7.0

Batoi Press 1.7.0 completes the latest testing-team backlog and adds safer, clearer authoring for mathematics, code samples, and layout-heavy special pages.

## Highlights

- Add hierarchical pages and nested menus with parent selection, stable child routes, active navigation states, draft exclusion, and static-export coverage.
- Improve post authoring with editable slugs and subtitles, featured-image URL handling, and previous/next article navigation.
- Support multiple media uploads with per-file validation feedback and reusable public asset URLs.
- Add editor actions for inline and display LaTeX, with conditional MathJax loading on public pages.
- Render semantic code blocks with distinct block styling, language labels, horizontal overflow handling, and a Copy action.
- Add protected text-only editing for layout-heavy special pages so authors can update visible copy without damaging raw HTML structure.
- Improve active-theme handling, expose the Contact layout in the constrained theme editor, and make the Recent Posts widget sortable with a configurable item limit.
- Accept standard and single-folder desktop update ZIPs while rejecting ambiguous package layouts and unsafe target collisions.

## Compatibility and Safety

- Existing flat page and post routes remain valid. Pages without a parent retain their current public URLs.
- Existing HTML content remains compatible. MathJax loads only when supported LaTeX delimiters are present, and code-block enhancements require no content migration.
- Text-only special-page saves are rejected when the protected source changes concurrently, preventing stale edits from overwriting newer layout work.
- Release packages preserve installation content and configuration by applying only manifest-listed installable files.

## Verification

- PHP syntax checks and the complete repository regression suite pass.
- Release packaging, manifest checksum, installable-file inventory, and runtime-state exclusions pass verification.
- The 1.7.0 update package is applied to the synchronized testsite through the normal staged updater path and browser-reviewed.
