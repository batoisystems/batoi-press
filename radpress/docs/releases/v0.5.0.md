# Batoi Press 0.5.0

Batoi Press 0.5.0 completes the next operational-hardening slice for pre-1.0 use: searchable admin lists, release artifact verification, and documented package-trust metadata groundwork.

## What Changed

- Pages now support bookmarkable search and status filters.
- Posts now support bookmarkable search and status filters across title, slug, taxonomy, and SEO metadata.
- Media now supports filename search, image/document filtering, and deterministic sort choices.
- Release manifests include disabled-by-default trust metadata for future signed-package support.
- Release artifacts can be verified locally with `php tools/verify-release-artifacts.php` before publication.

## Compatibility

This release is backward compatible with the v0.4.x content format, theme templates, public routes, installer layout, and update package model.

Unsigned release packages remain supported. The new trust metadata is informational in v0.5.0 and does not require package signatures.

## Release Checklist

- Update `radpress/config/update.json` to `0.5.0`.
- Run `php radpress/tests/smoke.php`.
- Run `php radpress/tests/update_runner.php`.
- Build the package with `php tools/build-release.php`.
- Generate the manifest with `php tools/generate-release-manifest.php`.
- Verify release outputs with `php tools/verify-release-artifacts.php`.
- Publish `dist/latest.json` and `dist/batoi-press-0.5.0.zip` to `https://www.batoi.com/pub/press/`.

## Follow-Up

Continue toward `1.0.0` by browser-reviewing the full admin console on desktop and mobile, expanding update package tests where needed, and deciding when signed-package enforcement should move from optional metadata to an enabled policy.
