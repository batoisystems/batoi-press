# Batoi Press 1.0.0

Batoi Press 1.0.0 is the first stable release of the documented flat-file CMS and publishing baseline.

## Stable Scope

- PHP 8.1+ runtime with no required database, Composer, Node.js, Docker, Git, or CLI access for normal hosting.
- FTP/cPanel-friendly deployment using browser-facing `public_html/` and private `radpress/` application files.
- Flat-file content storage for pages, posts, media, menus, settings, users, themes, audit logs, backups, cache, exports, and update staging.
- Authenticated admin workflows for dashboard, pages, posts, media, menus, settings, themes, users, audit log, cache, static export, updates, and Batoi AIF status.
- Stable public routes for pages, blog, post detail, media, sitemap, and feed output.
- Release ZIP and `latest.json` manifest generation with checksum verification.

## Compatibility Commitments

The `1.0.0` release treats these contracts as stable for normal production use:

- Page and post content directories with `meta.json` and `body.html`.
- Public route behavior for pages, posts, blog, media, sitemap, and feed.
- Theme templates under `radpress/theme/`.
- Browser installer layout and `radpress/config/installed.lock` lock behavior.
- Update package staging and manifest-listed file apply model.
- Release artifact publication under `https://www.batoi.com/pub/press/`.

Future `1.x` releases should preserve these contracts unless release notes document a migration.

## Verification

The stable release gate includes:

- `php radpress/tests/smoke.php`
- `php radpress/tests/update_runner.php`
- `php radpress/tests/static_export.php`
- `php radpress/tests/role_access.php`
- `php radpress/tests/post_ownership.php`
- `php radpress/tests/security_baseline.php`
- `php tools/build-release.php`
- `php tools/generate-release-manifest.php`
- `php tools/verify-release-artifacts.php`

## Deferred Post-1.0 Items

These are explicitly deferred and do not block `1.0.0`:

- Mandatory signed release package enforcement.
- Optional automatic media copying inside static export packages.
- Per-page ownership workflows.
- Multi-stage editorial approvals.
- Browser matrix screenshots for every role and viewport as a formal release artifact.

## Publication

Publish `dist/latest.json` and `dist/batoi-press-1.0.0.zip` to the public Batoi Press release directory after local verification.
