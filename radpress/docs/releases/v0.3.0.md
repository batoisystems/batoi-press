# Batoi Press 0.3.0

Batoi Press 0.3.0 focuses on making the Admin Console business-ready for routine publishing, site operations, governance, and release maintenance.

## Highlights

- Professional Admin Console layout with Batoi Press branding, persistent top navigation, and icon-backed actions.
- Standardized left-sidebar navigation across associated admin pages.
- Enhanced Dashboard with operating guidance, content status, quick actions, workflow, recent content, and runtime readiness.
- Improved Pages and Posts list/editor workflows with standards, SEO panels, publishing guidance, and pre-publish checklists.
- Bundled and styled Batoi UIF HTML editor assets for page, post, and theme template editing.
- Media upload directory handling, upload policy guidance, public URL snippets, and audit logging.
- Theme management with activation, upload, preview, constrained template editing, snapshots, restore, and safety guidance.
- Website favicon upload while keeping the admin favicon fixed to the Batoi Press icon.
- Static Export and Cache pages with readiness, package/download history, and operational notes.
- Full Audit Log operations: pagination, search, filters, CSV/JSONL export, cleanup, and 90-day minimum retention.
- Batoi AIF status page with readiness, feature flags, trust boundary, setup requirements, and disabled-by-default controls.
- Admin page standardization pass across all left-sidebar sections and associated pages.

## Versioning

- `radpress/config/update.json` is incremented to `0.3.0`.
- Release package and public `latest.json` manifest should be generated from that version.

## Verification

- PHP syntax checks for edited admin controllers.
- `php radpress/tests/smoke.php`
- Representative admin render checks.
- Release package build and manifest generation should be run before publishing the tag.

## Follow-Up Before 1.0.0

Use `radpress/docs/release-management.md` as the release gate. The `1.0.0` checklist should be completed before declaring Batoi Press stable for normal production use.
