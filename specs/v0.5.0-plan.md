## Out of Scope for v0.5.0

- Mandatory signed package enforcement.
- Mandatory database storage.
- Mandatory Composer, Node.js, Git, or command-line runtime requirements.
- Breaking changes to content files, theme templates, installer behavior, public routes, or update package structure.
- Broad admin redesign beyond the list-filter and release-safety surfaces.

## Verification Plan

- Run `php -l` on changed PHP files.
- Run `php radpress/tests/smoke.php`.
- Run `php radpress/tests/update_runner.php`.
- Run release artifact verification checks.
- Run `php tools/build-release.php`.
- Run `php tools/generate-release-manifest.php`.
- Verify `dist/latest.json` and `dist/batoi-press-0.5.0.zip` before copying to `batoi-www`.
- Browser-check `/admin/pages`, `/admin/posts`, and `/admin/media` filters when the testsite is updated.

## Recommended First Implementation Slice

Start with admin list filters. They close the most visible v0.4.0 follow-up, are contained to existing controllers, and provide immediate operational value before release safety and publication work.
